## What is Bun?

Bun is an **all-in-one JavaScript runtime** that includes:

1. **Runtime**: Executes JavaScript/TypeScript (like Node.js)
2. **Package manager**: Installs dependencies (like npm/yarn/pnpm)
3. **Bundler**: Bundles code for production (like webpack/esbuild)
4. **Test runner**: Runs tests (like Jest/Vitest)

### Bun vs Node.js

| Feature | Node.js | Bun |
|---------|---------|-----|
| Language | C++ | Zig |
| JS Engine | V8 | JavaScriptCore (Safari's engine) |
| TypeScript | Requires transpilation | Native support |
| Package manager | npm (separate) | Built-in |
| Speed | Fast | Very fast |
| Node.js compatibility | N/A | High (drop-in replacement) |

### Key Differences

```
Node.js workflow:
  1. Write TypeScript
  2. Compile to JavaScript (tsc)
  3. Run JavaScript (node)

Bun workflow:
  1. Write TypeScript
  2. Run TypeScript directly (bun)
```

## Why opencode Uses Bun

From the project's configuration and codebase, opencode chose Bun for:

### 1. Native TypeScript Execution

No build step needed during development:

```bash
# Run TypeScript directly
bun run src/index.ts

# Compare to Node.js which would need:
# tsc && node dist/index.js
```

### 2. Fast Package Installation

Bun's package manager is significantly faster than npm:

```bash
# Install all dependencies
bun install

# Add a new package
bun add zod
```

### 3. Built-in APIs

Bun provides optimized APIs that are faster than Node.js equivalents:

```typescript
// Bun's file API - simpler and faster
const content = await Bun.file("config.json").text()

// vs Node.js
import { readFile } from "fs/promises"
const content = await readFile("config.json", "utf-8")
```

### 4. Compilation to Single Binary

opencode compiles to standalone executables using `Bun.build`:

```typescript
await Bun.build({
  compile: {
    outfile: "dist/opencode",
    target: "bun-linux-x64"
  },
  entrypoints: ["./src/index.ts"]
})
```

## Bun APIs Used in opencode

### Bun.file() - Reading Files

`Bun.file()` creates a lazy file reference. The file isn't read until you call a method:

```typescript
// Create a file reference
const file = Bun.file("package.json")

// Read as text
const text = await file.text()

// Read as JSON
const json = await file.json()

// Read as ArrayBuffer
const buffer = await file.arrayBuffer()

// Get file metadata
console.log(file.size)  // File size in bytes
console.log(file.type)  // MIME type
```

**Real example** from `packages/opencode/script/build.ts`:

```typescript
// Read models data from file or fetch from URL
const modelsData = process.env.MODELS_DEV_API_JSON
  ? await Bun.file(process.env.MODELS_DEV_API_JSON).text()
  : await fetch(`${modelsUrl}/api.json`).then((x) => x.text())

// Read migration SQL files
const sql = await Bun.file(file).text()
```

### Bun.write() - Writing Files

```typescript
// Write text to a file
await Bun.write("output.txt", "Hello, World!")

// Write JSON
await Bun.write("config.json", JSON.stringify({ port: 4096 }, null, 2))

// Write to a Bun.file reference
await Bun.file("output.txt").write("Hello!")
```

**Real example** from `packages/opencode/script/build.ts`:

```typescript
// Generate TypeScript file with embedded data
await Bun.write(
  path.join(dir, "src/provider/models-snapshot.js"),
  `// Auto-generated by build.ts - do not edit
export const snapshot = ${modelsData}
`,
)

// Write package.json for distribution
await Bun.file(`dist/${name}/package.json`).write(
  JSON.stringify({
    name,
    version: Script.version,
    os: [item.os],
    cpu: [item.arch],
  }, null, 2),
)
```

### Bun.spawn() - Running Processes

```typescript
// Simple command
const proc = Bun.spawn(["ls", "-la"])
await proc.exited  // Wait for completion

// With options
const proc = Bun.spawn(["npm", "test"], {
  cwd: "/path/to/project",
  env: { ...process.env, NODE_ENV: "test" },
  stdout: "inherit",  // Show output in terminal
  stderr: "inherit",
})

// Capture output
const proc = Bun.spawn(["echo", "hello"])
const output = await new Response(proc.stdout).text()
```

**Real example** from `packages/app/script/e2e-local.ts`:

```typescript
// Start the opencode server
const server = Bun.spawn(
  ["bun", "run", "--conditions=browser", "./src/index.ts", "serve", "--port", port],
  {
    cwd: opencodeDir,
    stdout: "inherit",
    stderr: "inherit",
    env: {
      ...process.env,
      OPENCODE_DATA_HOME: dataDir,
    },
  },
)

// Wait for process to exit
const exitCode = await server.exited
```

### Bun.serve() - HTTP Server

```typescript
const server = Bun.serve({
  port: 3000,
  fetch(req) {
    const url = new URL(req.url)
    
    if (url.pathname === "/") {
      return new Response("Hello!")
    }
    
    if (url.pathname === "/json") {
      return Response.json({ message: "Hello" })
    }
    
    return new Response("Not Found", { status: 404 })
  },
})

console.log(`Server running at ${server.url}`)
```

**Real example** from `packages/opencode/src/mcp/oauth-callback.ts`:

```typescript
server = Bun.serve({
  port,
  fetch(req) {
    const url = new URL(req.url)
    const code = url.searchParams.get("code")
    const state = url.searchParams.get("state")

    if (!code || !state) {
      return new Response(errorHtml("Missing code or state"), {
        headers: { "Content-Type": "text/html" },
      })
    }

    // Process OAuth callback...
    return new Response(successHtml, {
      headers: { "Content-Type": "text/html" },
    })
  },
})
```

### Bun.build() - Bundling and Compilation

```typescript
// Bundle for browser
const result = await Bun.build({
  entrypoints: ["./src/index.ts"],
  outdir: "./dist",
  target: "browser",
  minify: true,
})

// Compile to standalone executable
const result = await Bun.build({
  entrypoints: ["./src/index.ts"],
  compile: {
    outfile: "./dist/myapp",
    target: "bun-linux-x64",
  },
})
```

**Real example** from `packages/opencode/script/build.ts`:

```typescript
await Bun.build({
  conditions: ["browser"],
  tsconfig: "./tsconfig.json",
  plugins: [solidPlugin],
  compile: {
    autoloadBunfig: false,
    autoloadDotenv: false,
    target: "bun-linux-x64",
    outfile: `dist/${name}/bin/opencode`,
    execArgv: [`--user-agent=opencode/${Script.version}`],
  },
  entrypoints: ["./src/index.ts", parserWorker, workerPath],
  define: {
    OPENCODE_VERSION: `'${Script.version}'`,
    OPENCODE_MIGRATIONS: JSON.stringify(migrations),
  },
})
```

### Bun.sleep() - Async Delays

```typescript
// Wait for 1 second
await Bun.sleep(1000)

// Useful in tests or polling
while (!ready) {
  await Bun.sleep(100)
  ready = await checkStatus()
}
```

### $ (Shell Template Tag)

Bun provides a convenient shell template tag:

```typescript
import { $ } from "bun"

// Run shell commands
await $`rm -rf dist`
await $`mkdir -p dist/bin`

// Capture output
const version = await $`git describe --tags`.text()

// Chain commands
await $`npm run build && npm test`

// With variables (automatically escaped)
const dir = "my folder"
await $`ls ${dir}`  // Safe, handles spaces
```

**Real example** from `packages/opencode/script/build.ts`:

```typescript
import { $ } from "bun"

await $`rm -rf dist`
await $`mkdir -p dist/${name}/bin`
await $`bun install --os="*" --cpu="*" @opentui/core@${pkg.dependencies["@opentui/core"]}`

// Create archives for release
await $`tar -czf ../../${key}.tar.gz *`.cwd(`dist/${key}/bin`)
await $`gh release upload v${Script.version} ./dist/*.zip --clobber`
```

## Running TypeScript Directly

One of Bun's biggest advantages is running TypeScript without compilation:

```bash
# Run a TypeScript file
bun run src/index.ts

# Or just
bun src/index.ts

# Run with watch mode (restarts on changes)
bun --watch src/index.ts
```

### How It Works

Bun transpiles TypeScript on-the-fly using its built-in transpiler. It:

1. Reads the `.ts` file
2. Strips type annotations (types are checked by your editor/IDE)
3. Executes the resulting JavaScript

**Note**: Bun does NOT type-check at runtime. Use `tsc --noEmit` or your editor for type checking.

## Package Management with Bun

### Installing Dependencies

```bash
# Install all dependencies from package.json
bun install

# Install with specific options
bun install --frozen-lockfile  # CI mode, fail if lockfile needs update
```

### Adding Packages

```bash
# Add a runtime dependency
bun add zod

# Add a dev dependency
bun add -d typescript

# Add a specific version
bun add zod@3.22.0

# Add from workspace
bun add @opencode-ai/sdk@workspace:*
```

### Removing Packages

```bash
bun remove zod
```

### Running Scripts

```bash
# Run a script from package.json
bun run build
bun run test

# Shorthand for common scripts
bun test
bun dev
```

## The bunfig.toml File

Bun uses `bunfig.toml` for configuration. Here's the opencode configuration from the repo root:

```toml
[install]
exact = true

[test]
root = "./do-not-run-tests-from-root"
```

### Configuration Explained

**`[install]`** - Package installation settings:

- `exact = true`: Install exact versions (no `^` or `~` prefixes)
  - Without this: `bun add zod` → `"zod": "^3.22.0"`
  - With this: `bun add zod` → `"zod": "3.22.0"`

**`[test]`** - Test runner settings:

- `root = "./do-not-run-tests-from-root"`: Points to a non-existent directory
  - This is a **guard** to prevent running tests from the repo root
  - Tests should be run from individual package directories instead

### Why This Guard Exists

From the AGENTS.md:

> Tests cannot run from repo root (guard: `do-not-run-tests-from-root`); run from package dirs like `packages/opencode`.

This ensures:
1. Tests run in the correct context
2. Package-specific test configurations are used
3. Isolation between package test suites

### Other bunfig.toml Options

```toml
[install]
exact = true
production = false  # Include devDependencies
frozen = false      # Allow lockfile updates

[test]
coverage = true     # Enable code coverage
timeout = 30000     # Test timeout in ms

[run]
# Environment variables for bun run
[run.env]
NODE_ENV = "development"
```
