## Message Architecture Overview

Messages in opencode follow a **part-based architecture**. Each message contains multiple parts that represent different types of content:

```
┌─────────────────────────────────────────────────────────────┐
│                      MESSAGE                                 │
├─────────────────────────────────────────────────────────────┤
│  id: message_01HWXYZ...                                     │
│  role: "user" | "assistant"                                 │
│  sessionID: session_abc123                                  │
│                                                             │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐       │
│  │ TextPart │ │ FilePart │ │ ToolPart │ │ReasonPart│  ...  │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘       │
│                                                             │
│  time.created: 1710000000000                                │
│  cost: 0.0012                                               │
│  tokens: { input: 1500, output: 200, ... }                  │
└─────────────────────────────────────────────────────────────┘
```

## Converting to Model Messages

The `toModelMessages` function transforms opencode's message format to the AI SDK format:

```typescript
// From packages/opencode/src/session/message-v2.ts

export function toModelMessages(
  input: WithParts[],
  model: Provider.Model,
  options?: { stripMedia?: boolean },
): ModelMessage[] {
  const result: UIMessage[] = []
  const toolNames = new Set<string>()
  
  // Handle media in tool results based on provider support
  const supportsMediaInToolResults = (() => {
    if (model.api.npm === "@ai-sdk/anthropic") return true
    if (model.api.npm === "@ai-sdk/openai") return true
    if (model.api.npm === "@ai-sdk/amazon-bedrock") return true
    // ... more providers
    return false
  })()

  for (const msg of input) {
    if (msg.parts.length === 0) continue

    if (msg.info.role === "user") {
      const userMessage: UIMessage = {
        id: msg.info.id,
        role: "user",
        parts: [],
      }
      result.push(userMessage)
      
      for (const part of msg.parts) {
        if (part.type === "text" && !part.ignored)
          userMessage.parts.push({
            type: "text",
            text: part.text,
          })
        
        if (part.type === "file" && part.mime !== "text/plain") {
          if (options?.stripMedia && isMedia(part.mime)) {
            userMessage.parts.push({
              type: "text",
              text: `[Attached ${part.mime}: ${part.filename ?? "file"}]`,
            })
          } else {
            userMessage.parts.push({
              type: "file",
              url: part.url,
              mediaType: part.mime,
              filename: part.filename,
            })
          }
        }

        if (part.type === "compaction") {
          userMessage.parts.push({
            type: "text",
            text: "What did we do so far?",
          })
        }
      }
    }

    if (msg.info.role === "assistant") {
      // Skip errored messages (unless aborted with content)
      if (msg.info.error && !(
        MessageV2.AbortedError.isInstance(msg.info.error) &&
        msg.parts.some((part) => part.type !== "step-start" && part.type !== "reasoning")
      )) {
        continue
      }
      
      const assistantMessage: UIMessage = {
        id: msg.info.id,
        role: "assistant",
        parts: [],
      }
      
      for (const part of msg.parts) {
        if (part.type === "text")
          assistantMessage.parts.push({
            type: "text",
            text: part.text,
          })
          
        if (part.type === "tool") {
          toolNames.add(part.tool)
          
          if (part.state.status === "completed") {
            const outputText = part.state.time.compacted 
              ? "[Old tool result content cleared]" 
              : part.state.output
            // ... handle attachments
            
            assistantMessage.parts.push({
              type: ("tool-" + part.tool) as `tool-${string}`,
              state: "output-available",
              toolCallId: part.callID,
              input: part.state.input,
              output,
            })
          }
          
          if (part.state.status === "error")
            assistantMessage.parts.push({
              type: ("tool-" + part.tool) as `tool-${string}`,
              state: "output-error",
              toolCallId: part.callID,
              input: part.state.input,
              errorText: part.state.error,
            })
            
          // Handle interrupted tools
          if (part.state.status === "pending" || part.state.status === "running")
            assistantMessage.parts.push({
              type: ("tool-" + part.tool) as `tool-${string}`,
              state: "output-error",
              toolCallId: part.callID,
              input: part.state.input,
              errorText: "[Tool execution was interrupted]",
            })
        }
        
        if (part.type === "reasoning") {
          assistantMessage.parts.push({
            type: "reasoning",
            text: part.text,
          })
        }
      }
      
      if (assistantMessage.parts.length > 0) {
        result.push(assistantMessage)
      }
    }
  }

  // Convert to AI SDK format
  return convertToModelMessages(
    result.filter((msg) => msg.parts.some((part) => part.type !== "step-start")),
    { tools }
  )
}
```

### Key Conversion Rules

| opencode Part | AI SDK Part |
|---------------|-------------|
| `TextPart` | `{ type: "text", text }` |
| `FilePart` (media) | `{ type: "file", url, mediaType }` |
| `ToolPart` (completed) | `{ type: "tool-{name}", state: "output-available" }` |
| `ToolPart` (error) | `{ type: "tool-{name}", state: "output-error" }` |
| `ToolPart` (interrupted) | `{ type: "tool-{name}", state: "output-error", errorText: "[interrupted]" }` |
| `ReasoningPart` | `{ type: "reasoning", text }` |
| `CompactionPart` | `{ type: "text", text: "What did we do so far?" }` |

## Handling Interrupted Tools

When a session is aborted or resumed, some tools may be in `pending` or `running` state. The conversion handles this:

```typescript
// Handle pending/running tool calls to prevent dangling tool_use blocks
// Anthropic/Claude APIs require every tool_use to have a corresponding tool_result
if (part.state.status === "pending" || part.state.status === "running")
  assistantMessage.parts.push({
    type: ("tool-" + part.tool) as `tool-${string}`,
    state: "output-error",
    toolCallId: part.callID,
    input: part.state.input,
    errorText: "[Tool execution was interrupted]",
  })
```

This ensures the model knows the tool didn't complete and can retry if needed.

## Message Pagination

Messages are paginated for efficient loading:

```typescript
export const page = fn(
  z.object({
    sessionID: SessionID.zod,
    limit: z.number().int().positive(),
    before: z.string().optional(),     // Cursor for pagination
  }),
  async (input) => {
    const before = input.before ? cursor.decode(input.before) : undefined
    const where = before
      ? and(eq(MessageTable.session_id, input.sessionID), older(before))
      : eq(MessageTable.session_id, input.sessionID)
      
    const rows = Database.use((db) =>
      db
        .select()
        .from(MessageTable)
        .where(where)
        .orderBy(desc(MessageTable.time_created), desc(MessageTable.id))
        .limit(input.limit + 1)  // Fetch one extra to check for more
        .all(),
    )
    
    const more = rows.length > input.limit
    const page = more ? rows.slice(0, input.limit) : rows
    const items = await hydrate(page)  // Load parts for each message
    items.reverse()
    
    return {
      items,
      more,
      cursor: more && tail ? cursor.encode({ id: tail.id, time: tail.time_created }) : undefined,
    }
  },
)
```

### Cursor Encoding

```typescript
export const cursor = {
  encode(input: Cursor) {
    return Buffer.from(JSON.stringify(input)).toString("base64url")
  },
  decode(input: string) {
    return Cursor.parse(JSON.parse(Buffer.from(input, "base64url").toString("utf8")))
  },
}
```

## Message Streaming

For real-time loading, messages can be streamed:

```typescript
export const stream = fn(SessionID.zod, async function* (sessionID) {
  const size = 50
  let before: string | undefined
  while (true) {
    const next = await page({ sessionID, limit: size, before })
    if (next.items.length === 0) break
    
    // Yield in reverse order (oldest first)
    for (let i = next.items.length - 1; i >= 0; i--) {
      yield next.items[i]
    }
    
    if (!next.more || !next.cursor) break
    before = next.cursor
  }
})
```

## Filtering Compacted Messages

When loading messages, we filter to only show content after the last compaction:

```typescript
export async function filterCompacted(stream: AsyncIterable<MessageV2.WithParts>) {
  const result = [] as MessageV2.WithParts[]
  const completed = new Set<string>()
  
  for await (const msg of stream) {
    result.push(msg)
    
    // Stop at compaction boundary
    if (
      msg.info.role === "user" &&
      completed.has(msg.info.id) &&
      msg.parts.some((part) => part.type === "compaction")
    )
      break
      
    // Track completed summaries
    if (msg.info.role === "assistant" && msg.info.summary && msg.info.finish && !msg.info.error)
      completed.add(msg.info.parentID)
  }
  
  result.reverse()
  return result
}
```

## Error Handling

Errors are converted to a standardized format:

```typescript
export function fromError(
  e: unknown,
  ctx: { providerID: ProviderID; aborted?: boolean },
): NonNullable<Assistant["error"]> {
  switch (true) {
    case e instanceof DOMException && e.name === "AbortError":
      return new MessageV2.AbortedError({ message: e.message }).toObject()
      
    case MessageV2.OutputLengthError.isInstance(e):
      return e
      
    case LoadAPIKeyError.isInstance(e):
      return new MessageV2.AuthError({
        providerID: ctx.providerID,
        message: e.message,
      }).toObject()
      
    case APICallError.isInstance(e):
      const parsed = ProviderError.parseAPICallError({
        providerID: ctx.providerID,
        error: e,
      })
      if (parsed.type === "context_overflow") {
        return new MessageV2.ContextOverflowError({
          message: parsed.message,
        }).toObject()
      }
      return new MessageV2.APIError({
        message: parsed.message,
        statusCode: parsed.statusCode,
        isRetryable: parsed.isRetryable,
      }).toObject()
      
    default:
      return new NamedError.Unknown({ message: String(e) }).toObject()
  }
}
```

## Self-Check Questions

1. **What's the difference between `synthetic` and `ignored` on a TextPart?**
   <details>
   <summary>Answer</summary>
   `synthetic` means the text was generated by the system (not user input). `ignored` means the text is excluded from the model context entirely.
   </details>

2. **How are interrupted tool calls handled when converting to model messages?**
   <details>
   <summary>Answer</summary>
   They're converted to error results with the message "[Tool execution was interrupted]" to satisfy the API requirement that every tool_use has a tool_result.
   </details>

3. **What does `time.compacted` on a ToolStateCompleted indicate?**
   <details>
   <summary>Answer</summary>
   It indicates when the tool output was cleared during compaction. When set, the output is replaced with "[Old tool result content cleared]" in the model context.
   </details>

4. **Why does `filterCompacted` track `completed` assistant messages?**
   <details>
   <summary>Answer</summary>
   It tracks which user messages have completed summary responses, so it knows when to stop at a compaction boundary (where the summary is complete).
   </details>
